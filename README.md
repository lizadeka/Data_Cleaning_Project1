# 🧹 SQL Data Cleaning Project – Orders Dataset

## 📌 Objective
The goal of this project is to clean and prepare a messy orders dataset using SQL. The dataset contained multiple data quality issues that were identified and resolved to make it analysis-ready.

---

## 📂 Dataset Overview
The dataset includes customer order details such as:
- Customer Name
- Email
- City
- Order Date
- Order Amount

---

## ⚠️ Data Issues Identified
The following issues were found in the dataset:

- Duplicate records  
- Missing (NULL) email values  
- Invalid email formats  
- Extra spaces in customer names  
- Inconsistent city spellings  
- Multiple date formats  
- Missing order dates  
- Negative order amounts  
- Incorrect data types  

---

## 🛠️ Data Cleaning Steps

The cleaning process was performed using SQL queries:

1. Identified duplicate records  
2. Removed duplicate entries  
3. Detected NULL email values  
4. Replaced missing emails with default/valid values  
5. Identified invalid email formats  
6. Corrected invalid email entries  
7. Trimmed extra spaces in customer names  
8. Standardized city name spellings  
9. Removed extra spaces in city column  
10. Standardized multiple date formats into a consistent format  
11. Handled missing date values  
12. Corrected negative order amounts  
13. Replaced NULL order amounts  

---

## 🔄 Final Step
- Converted incorrect data types into appropriate SQL data types for consistency and accuracy  

---

## ✅ Outcome
- Cleaned and structured dataset ready for analysis  
- Improved data consistency and accuracy  
- Reduced errors in downstream analysis
